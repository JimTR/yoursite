NoIdeer Software

NoIdeer Software released it's first app in 1982 which was written in GW Basic and Tru Basic (the tru basic version followed in 1985), the idea of this app was to run in the DESQview enviroment this allowed the app to multi task with the next app that came from NoIdeer.
As time progressed NoIdeer moved it's coding language to Turbo Basic, but still using DESQview to multitask between modules of the app. During this period we wrote some add ons for the Merlin bbs system, written by Mark Lewis, which extended the forum and administration capabilities of Merlin.

In the early/mid 1990's NoIdeer 'moved on' to visual basic and continued to maintain the Merlin extensions with out the need for DESQview (as Microsoft had developed the 'windows' enviroment) and at the same time started to develop two core programes, the first being a TAPI application which (in it's day) simplified booking appointments to see a doctor and the second was a full EPOS system for small retailers that used CASIO cash registers (source code for both apps are availble if someone wants to develop these old apps to the present day, but will need converting from Microsoft Visual Basic 6)

In the 2000's NoIdeer Software produced 4 (minor) games for the windows desktop platform, using Visual Basic 6 as a coding language and then in 2011, moved into the 'online' application area using 'server side' languages as it's default modal. We did look at using Mybb as an application frame work, however after a period of time testing with mybb software it did not fit the requirements of NoIdeer and we developed our own. Our current product 'YourSite' has taken some ideas from mybb, however the vison is different, Yoursite is going to be a modular software platform where each section of a website has it's own settings and enviroment with one difference for users one login will access all modules

Code Ideas

YourSite uses two methods of producing variables for your templates

  • Static - these variables are added to a simple text file on your web server
  • Dynamic - these variables are created 'on the fly' by the module, there will be instruction (soon) on how to create a module
  • This allows the web site owner to add customised content to the site users
  • currently YourSite has no admin contol panel so all edits are through text files on the web server
